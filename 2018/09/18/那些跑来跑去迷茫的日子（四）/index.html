那些跑来跑去迷茫的日子（四）

Posted on 2018-09-18 | In 面试

神州数码信息技术有限公司（南京信息工程大学站）

认为自己的能力与理想不一致，我应该找工作还是考研？

  神州数码是我在南京9月3日投的简历，时隔两周之后邮件显示在线测评，可能是我太急了吧，我现在已经不在南京了，笔试题做了一部分，主要做了java开发的笔试题、逻辑笔试、性格笔试等，一共七部分，我完成了4部分。由于已经身不再南京，所以后面的几部分测试题直接放弃，关闭了浏览器。

突然之间想考研了。

部分选择题。

  1. 异常通常包括以下哪些内容。（）
    • 程序中的语法错误
    • 程序中的编译错误
    • 程序执行过程中遇到的事先没有预料到的情况
    • 程序中事先定义好的可能出现的意外情况
  2. 请阅读下列程序。输出结果是（）
    • 1
    • 3
    • 4
    • 什么都没输出

  1. 在java中，以下（）对象可以使用键/值的形式保存数据

    • LinkedList
    • ArrayList
    • HashMap
    • Arrays

     解析：键值对为Map的特点，HashMap继承了Map

  2. DataInputStream和DataOutputStream的特点（）

    • 读写数据时与具体的机器无关
    • 读写数据时与具体的机器有关
    • 读数据时与机器有关，而写数据时与机器无关
    • 读数据时与机器无关，而谢书记时与机器有关

     解析：InputStream和OutputStream是字节流，与目标机器无关，InputStreamReader和OutputStreamReader是字符流，与目标机器的软硬件环境有关。

  3. java语言中，下列哪一子句是异常处理的出口（）

    • catch{……}
    • try{……}
    • finally{……}
    • 以上说法都不对

  4. 下列数据类型进行运算时，哪一个会发生自动类型提升（）

    • int + int
    • long + long
    • byte + byte
    • double + double

     解析：byte运算时，会转换成int运算，这有别人写的一个例子

  5. java程序的执行过程中用到一套JDK工具，其中javac.exe指（）

    • java字节码解释器
    • java语言编译器
    • java文档生成器
    • java类分解器

     解析： java字节码解释器：java.exe，翻译class字节码问价；java语言编译器：javac.exe，用来将java源代码转换为class字节码；java文档生成器：javadoc.exe；java类分解器：javaP，用来分析java字节码

  6. Java中，在单一文件中import、class和package的正确出现顺序是（）

    • package,import,class
    • import,package,class
    • import,class,package
    • package,class,import

  7. 在Java中名字必须与它所在的类的名字相同，而且不返回任何数据类型的方法是（）

    • 构造方法
    • 类方法
    • 实例方法
    • 重载方法

  8. 凡是从中央处理器流向外部设备的数据流成为（）

    • 文件流
    • 字符流
    • 输入流
    • 输出流

  9. 下面关于对象串行化（Serializable）的说法不正确的是（）

    • 一个类实现接口Serializable就能使之串行化，但该接口没有具体方法需要实现
    • 一个对象串行化后，能通过对象流读取对象或写入对象
    • 用transient关键字修饰的变量将参与串行化
    • 串行化一个类，必须保证在恢复时java虚拟机能找到这个类的class文件，否则将会出现ClassNotFoundException

  10. 关于方法的递归，以下说法错误的是（）

    • 方法递归必须具有结束条件
    • 方法递归次数太多会导致内存溢出
    • 方法递归就是指在一个方法的内部调用自身的过程
    • 方法递归可以无数次，只要有结束条件就可以

    解析：递归的过程中使用了栈，若递归次数太多，内存容易溢出或者程序跑飞

  11. 线程同步中，对象的锁在（）情况下持有线程返回

    • 当synchronized（）语句块执行完后
    • 当在synchronized（）语句块执行中出现例外（Exception）时
    • 当持有锁的线程调用该对象的wait（）方法时
    • 以上都是

  12. 下面关于整型类型的描述中，错误的是（）

    • byte是java中的整数类型，它的取值范围是-127~128
    • java中的整数有正负之分
    • bit是java中的整数类型，它代表的是一个字节
    • int是java中的整数类型，它在内存中占4个字节

    解析：题有问题，因为是单选，但是byte范围在-128~127，bit不是java中的整数类型，而且他代表的是一个比特位，byte代表一个字节，答题的时候我选择了C

  13. 下面关于线程优先级的说法中，错误的一项是（）

    • MIN_PRIORITY代表最低优先级
    • MAX_PRIORITY代表最高优先级
    • NORM_PRIORITY代表普通优先级
    • 代表优先级的常数值越大优先级越低

    解析：

    • java并不是严格遵守线程优先级来执行的。Java是跨平台的系统,需要把10个优先级映射成不同操作系统的优先级,于是界定了Java的优先级只是代表抢占CPU的机会大小,数字越大优先级越高,抢占CPU的机会越大.被执行的可能性越高。于是推荐使用优先级常量，也就是前三个选项。如果优先级相同，基本上按照FIFO原则，但是也不能完全保证。详情点我
    • 在google books中的《PC技术真题分析与实战训练》写明了，在windows系统中，使用32个线程优先级（0~31），数字越大则优先级越高，具有最高优先级的就绪线程获取CPU的使用权，如果等待线程和执行线程有同样的优先权，系统会周期性地切换它们，让它们共享CPU。详情点我
    • 此题中应该指的是java环境中，所以选D

  14. 假设 int a=1,b=2,c=3，以下语句正确的是（）

    • c = c/float(a/b)
    • c = c/(float a/b)
    • c = (float)c/(a/b)
    • c = c/(int)(a/(float)b)

  15. 下列语句序列执行后，i 的值为（）

    • 15
    • 16
    • 7
    • 8

  16. 请阅读下面的程序，运行结果是（）

    • x=8,y=22
    • x=22,y=8
    • x=3,y=0
    • x=0,y=3

  17. list中定义了一个ArrayList的对象，哪个选项的代码放到//todo delete处，可以在Iterator遍历的过程中正确并安全的删除一个list中保存的对象？（）

    • list.remove(obj);
    • list.remove(index);
    • list.remove(it.next());
    • list.remove();

  18. 下面程序的执行结果是（）

    • 产生编译错误
    • 代码可以编译运行，并输出结果AB
    • 代码可以编译运行，但没有输出
    • 编译没有错误，运行时异常

编程题。

  1. 有一组N个数，想将数组进行从小到大排好序，但是只能通过下面的操作进行排序，任取数组中的一个数然后将它放置在数组的最后一个位置。问最少操作多少次可以使得数组从小到大进行有序？

    样例输入：19 2 4 16 10

    样例输出：2

  2. 现在有一棵合法的二叉树，树的结点都是用数字表示，现在给定这棵树上所有的父子关系。求这棵树的高度。

    输入的第一行表示节点的个数n（1<=n<=1000，节点的编号为0到n-1）组成，下面是n-1行，每行有两个整数，第一个数表示父节点的编号，第二个数表示子节点的编号。输出数的高度，为一个整数。

    样例输入：5
          0   1

          0   2

          1   3

          1   4
    样例输出：3

    （使用在线编译器，由于选做java，所以编程语言只有java，ctrl+i代码提示）
